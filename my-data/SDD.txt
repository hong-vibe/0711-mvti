새로운 진단 흐름(MBTI 선택 ➔ 영화 3개 선택 ➔ MVTI 도출 ➔ 4번 간격 해석 출력)에 맞추어, 제공해주신 **Deep Research 보고서의 인지적 선호 기제(N/S, T/F 등) 분석 결과**를 기반으로  16개 유형별 정밀 해석지를 작성했습니다.

이번 설계는 **클라이언트 사이드 인메모리 연산**으로 100% 구동되므로, 남은 시간 동안 별도의 DB 연동 없이 단일 `App.jsx` 파일로 초고속 빌드가 가능합니다.

# 1. PRD (Product Requirements Document)

## Overview

* **제품명:** MVTI (Movie + MBTI) 인터랙티브 싱글 페이지 진단 툴
* 
**목적:** 실제 성격(MBTI)과 영화 소비를 통해 드러나는 무의식적 취향 성격(MVTI)의 간극(Gap)을 학술적 기제에 기반해 분석해 주는 그로스 해킹형 웹앱.



## Key Features

* **MBTI & 영화 멀티 토글 (Step 1 & 2):** 유저가 본인의 MBTI를 고르고, 힌트가 숨겨진 영화 버튼 중 3개 이상을 선택.
* 
**실시간 인메모리 MVTI 연산 (Step 3):** 선택한 영화들의 MBTI 가중치를 즉시 계산하여 4글자 MVTI 조합.


* 
**심리인식형 간격 해석 (Step 4):** 오리지널 MBTI와 미디어 소비 성향의 심리적 연관성을 대조하여 맞춤형 텍스트 출력.



## 제외 범위 (Non-Goals)

* 30분 타임어택 스코프 유지를 위해 외부 백엔드 통신, TMDB API 연동, 유저 로그인(Auth)은 완전히 제외하며 오직 프론트엔드 정적 로직으로 완성함.



---

# 2. SRS (Software Requirements Specification)

## Functional Requirements

* **F-REQ-01 (영화 숨김 속성):** 영화 버튼 렌더링 시 제목만 노출해야 하며, 데이터 태그나 UI 상에 MBTI 매핑 정보를 절대 노출하지 않는다.
* **F-REQ-02 (MVTI 차원별 판정):** 선택된 영화들의 MBTI 빈도를 계산하되, 동점 발생 시 내향(I), 직관(N), 감정(F), 인식(P)에 우선순위를 둔다.
* 
**F-REQ-03 (해석지 매핑 정책):** 4번 해석지는 **유저가 1번에서 선택한 원래의 MBTI**를 기준으로 바인딩하여, 유저가 평소 지닌 성격적 특성과 영화 속에서 발현된 MVTI 취향 간의 심리적 보완/일치 관계를 설명한다.
