## 5\. 실행 방법

### 5-1. 설치 및 실행

윈도우 PowerShell의 스크립트 실행 제한 에러를 방어하기 위해 `.cmd` 확장자를 명시적으로 실행하는 것을 권장합니다.

```bash
# 1. 의존성 라이브러리 설치
npm.cmd install

# 2. 로컬 개발 서버 구동 (포트 3000번, 자동 웹 브라우저 실행)
npm.cmd run dev
```

### 5-2. 실행 확인 방법

1. 브라우저에서 실행 주소 `http://localhost:3000/`을 엽니다.
2. 메인 랜딩 포스터 슬라이드가 정상 구동되는지 검증하고 좋아하는 영화 3편, 싫어하는 영화 3편을 차례로 고릅니다.
3. 성격 유형(MBTI)을 입력하고 "결과 분석하기"를 탭해 최종 온보딩을 마칩니다.
4. 취향 진단서(`/result`)와 영화 추천 대시보드(`/dashboard`)에서 맞춤 추천 카드 내용이 노출되는지 확인합니다.
5. 메뉴의 "취향 기록 (CRUD)" 페이지로 넘어가 수동 반응 추가 창을 확인합니다.
6. 우측 검색창에서 영화를 검색한 후 선호도, 강도, 감상평 메모를 기재하여 "기록 저장하기"로 카드를 추가합니다.
7. 반응 카드의 연필 아이콘을 클릭하여 수정 모달창을 띄운 후 점수를 변경해 봅니다.
8. 반응 카드의 쓰레기통 버튼을 눌러 커스텀 다이얼로그를 호출하여 삭제 연산을 적용해 봅니다.
9. 브라우저를 새로고침(F5)하여 수정한 모든 CRUD 내역이 풀리지 않고 온전히 유지되는지 검증합니다.

## 16\. 배운 점

1. **리액트 상태의 불변성 수호**: 리액트 배열 조작 시 splice나 push 같은 직접 훼손 함수를 피하고 map, filter 등을 사용해 항상 독립적 새 인스턴스로 복사해야 UI 가 유기적으로 반응한다는 핵심 법칙을 배웠습니다.
2. **에러 핸들러와 폴백 구조**: 브라우저 로컬스토리지 용량 초과나 파싱 깨짐이 발생할 때 애플리케이션 자체가 멈추지 않도록 catch하여 초기 엠프티 값으로 복구하는 방어적 훅 아키텍처의 가치를 배웠습니다.
3. **요구사항 우선 설계**: 코드를 쳐나가기 전 데이터 모델과 사용자의 한 줄 요건을 먼저 명확히 테이블로 나누어 설계해 두는 것이 AI 협업 바이브 코딩에서 최단 속도로 버그 없는 앱을 만드는 기초임을 배웠습니다.
