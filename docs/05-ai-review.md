# AI 생성 결과 검토표 (`05-ai-review.md`)

본 검토표는 AI가 작성하고 제안한 파일 구조, 컴포넌트, 비즈니스 로직에 대해 설계 지침 및 보안 수칙 준수 여부를 개발자가 직접 검토한 결과 기록입니다.

---

## 1. AI 생성 결과 자가 검토 결과표

| 검토 항목 | 확인 질문 | 검토 결과 | 비고 / 수정 조치 내역 |
| --- | --- | --- | --- |
| **요구사항 충족** | 목록 조회, 추가, 수정, 삭제 기능이 모두 고려되었나요? | **예** | `useReactions` 훅에 CRUD 핵심 핸들러가 모두 탑재됨. |
| **데이터 구조** | id와 필수 필드가 명확하게 설계되었나요? | **예** | UUID 형태의 id와 외래키 movieId, sentiment 등 상세 구조 매핑 완료. |
| **파일 구조** | 파일 역할이 스펙에 맞게 독립적으로 분리되었나요? | **예** | components, hooks, utils, data 폴더로 명확히 역할 분리. |
| **컴포넌트 구조** | 컴포넌트의 역할 분배와 props 전달이 깔끔한가요? | **예** | App(상태 관리) ➔ 하위 프레젠테이션 컴포넌트로 단방향 props 전달. |
| **상태 관리** | React 상태 업데이트 시 기존 배열을 직접 변경하지 않나요? | **예** | push/splice 등을 금지하고 `[...prev]`, `map`, `filter`로 불변성 준수. |
| **입력 검증** | 빈 값 제출이나 중복 영화 추가에 대해 적절한 유효성 체크를 하나요? | **예** | sentiment 누락 가드 및 중복 영화 추가 시 에러 발생 가드 탑재. |
| **빈 상태 (Empty)** | 목록에 데이터가 없을 때 보완용 렌더링 메시지가 제공되나요? | **예** | SelectionTray 등에 데이터가 비어 있을 시의 친절한 가이드 노출. |
| **저장 구조** | LocalStorage 연동 구조가 에러 핸들링과 함께 명확히 설계되었나요? | **예** | `useLocalStorage` 훅에 try-catch 예외 파싱 복구 루틴 추가. |
| **보안 점검** | API Key, .env가 코드나 저장소에 노출되지 않도록 처리되었나요? | **예** | `.gitignore`에 `.env` 및 `node_modules` 배포 제외 지정 완료. |
| **과도한 구현** | 과제 범위를 벗어난 복잡한 프레임워크나 외부 라이브러리를 배제했나요? | **예** | Redux/Recoil, Tailwind 등을 배제하고 순수 Hooks와 Vanilla CSS로 제한. |
| **README** | 컴포넌트 구조, 데이터 모델, 실행 방법이 README에 준비되었나요? | **예** | `README.md` 내에 실행 방법 및 아키텍처 가이드라인 작성 예정. |

## 2. AI 코드를 검토하여 직접 보정한 사항 (주요 수동 디버깅 내역)

1. **영화 데이터 카테고리 불균형 보정**:
   - AI가 생성한 `fetchMovies.mjs` 시드 데이터 중 `어벤져스`, `헤어질 결심`, `소울`, `오펜하이머` 등이 카테고리별 장르 수량 불일치(12개씩 딱 안 맞음)를 야기하는 것을 인지하고, primaryGenre 문자열 속성값을 드라마/SF/코미디 등에 맞게 1대1 매핑하여 8개 장르별 정확히 12편씩의 균형을 유지하도록 수정하였습니다.
2. **그래비티 TMDB ID 매핑 충돌 버그 해결**:
   - 기존 시드 중 `그래비티` 영화의 TMDB ID가 `116745`(월터의 상상은 현실이 된다 ID)로 잘못 겹쳐 적혀 있는 것을 발견하고, 그래비티 고유의 TMDB ID인 `12477`로 바르게 정정하여 영화 수집 데이터가 꼬이는 중대한 결함을 디버그했습니다.
3. **5편의 신규 영화 추가 확장**:
   - 96편 명작 풀을 훼손하지 않기 위해, 기존 96편을 롤백하여 온전히 보존한 뒤 사용자가 요청한 영화 5편을 씨드 배열 맨 끝에 Append하는 방식으로 안정성을 강화하였습니다.
